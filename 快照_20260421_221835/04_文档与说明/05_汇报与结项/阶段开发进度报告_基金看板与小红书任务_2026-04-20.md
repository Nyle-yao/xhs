# 阶段开发进度报告：基金加减仓看板与小红书数据挖掘任务



面向：温姐、芬姐｜更新时间：2026-04-20 21:29



## 一、整体定位

目前我在推进两条互相关联但相对独立的数据工作线：一条是“基金加仓减仓榜看板”，用于日常观察支付宝等互联网渠道中基金产品的加仓、减仓、持续上榜和当日排序情况；另一条是“小红书数据挖掘任务”，用于观察财经/基金类达人内容中出现的基金提及、疑似投放、评论反馈和用户偏好信号。

这两项工作的共同目标不是单纯做表格或爬数据，而是逐步形成可复查、可更新、可解释的数据辅助工具，帮助互联网金融运营更快识别市场风向、竞品动作和内容机会。



## 二、本阶段已经完成的主要工作

1. 基金加仓减仓榜看板方面：核心看板已经形成稳定版本，支持加仓表、减仓表、双边表和当日排序表。当前核心看板可以按日期查看历史数据，也可以按基金范围和基金类型筛选，帮助快速判断某一天或某一段时间内哪些基金持续上榜。

2. 当日排序表已做关键补强：增加了“全部基金、偏股基金、偏债基金、QD基金”的范围筛选，同时增加基金类型筛选，避免过去“类型”字段全部显示为同一个值、信息价值不高的问题。

3. 数据准确性方面：对核心看板做了全历史校验，不只检查当天，而是检查历史日期、加仓/减仓、不同基金范围下的数据是否一致，避免出现本地正确、网页不正确，或切换日期后数据不联动的问题。

4. 新手导航页已补充历史迭代说明，便于运营同事了解看板怎么用、每次迭代修复了什么、遇到数据疑问时如何判断。

5. 小红书任务方面：已根据目标博主名单完成一轮抓取流程验证，产出博主信息、笔记信息、评论信息、基金提及、失败记录等基础表。

6. 针对前一版小红书数据“表格较乱、有效信息不清晰、基金识别不足”的问题，已新增一版质量诊断与运营分析输出，将原始抓取结果拆解成博主质量总览、笔记质量诊断、评论质量诊断、确认基金运营汇总、候选基金词待确认、优先补抓清单等更容易理解的表。



## 三、已发现并处理的问题

1. 看板日期联动问题：曾发现切换日期后，部分涨跌幅字段没有跟着日期变化。已调整为“先选日期，再看该日期对应数据”的强联动口径。

2. 基金代码格式问题：曾发现基金代码前导 0 容易丢失。当前看板和小红书分析表均按文本格式处理基金代码，避免把 007818 显示成 7818 这类问题。

3. 看板本地与网页不同步问题：追平网页时已增加线上复核步骤，确认不是只部署成功，而是网页数据、筛选逻辑和本地版本一致。

4. 小红书数据质量问题：前一版抓取结果里存在大量占位内容、评论覆盖不足、图片信息不足、基金提及识别偏少等问题。本阶段没有简单继续堆数据，而是先建立质量诊断机制，把“哪些数据可用、哪些需要补抓、哪些只是候选词”先分清楚。

5. 基金识别误判问题：曾出现“股票基金、ETF基金、黄金基金”等泛词被纳入基金汇总的风险。已将“确认基金”和“候选基金词”拆开，确认基金用于正式分析，候选基金词用于人工复核和后续别名库补充。



## 四、当前产出材料

1. 基金加仓减仓核心看板：已上线，可用于日常查看加仓、减仓、双边和当日排序。

2. 新手导航页：已上线，用于给运营同事说明看板入口、使用方式、常见疑问和历史迭代修复。

3. 小红书质量诊断与运营分析表：已生成当前版本，主要用于判断小红书抓取结果的可用性和下一步补抓重点。

4. 小红书算法升级记录：已整理本次升级背景、解决问题、当前局限和下一步计划，便于后续汇报和交接。



## 五、当前困难与处理方式

1. 小红书平台本身反爬较强，登录态、页面加载、评论展开、图片内容识别都会影响抓取完整性。当前处理方式是先建立分层诊断，不盲目承诺一次性全量抓准，而是把失败原因和待补抓对象单独列出。

2. 小红书内容里基金名称可能被简称、化名、图片文字、评论补充等形式表达，单靠基金全称匹配不够。当前处理方式是使用基金名称、基金代码、别名和候选词识别，并将不确定内容放入候选表等待人工确认。

3. 小红书图片 OCR 和评论数据仍是难点。当前已经把图片缺失、占位内容、评论未抓到等问题放入优先补抓清单，后续会优先解决高价值博主和高基金语义笔记。

4. 数据表如果只按技术导出，会让业务同事看不懂。当前已改为面向运营使用的结构，先看总览，再看博主质量、笔记质量、基金确认结果和补抓清单。



## 六、阶段性结论

基金加仓减仓核心看板目前已经进入相对稳定可用阶段，可以作为日常观察同业基金热度和用户加减仓风向的主入口。

小红书任务目前还没有到最终成品阶段，但已经从“能抓一些数据”推进到“能判断数据质量、能区分确认基金和候选基金词、能明确下一步补抓对象”的阶段。

下一步小红书任务的重点不是继续无差别扩大抓取量，而是围绕高价值博主、高基金语义笔记和评论证据链做定向补抓与算法增强。



## 七、下一步计划

1. 继续优化小红书评论抓取，优先抓博主本人评论、高赞评论和含基金关键词的评论。

2. 建立基金化名/简称人工确认机制，把候选基金词中确认有效的内容回灌到别名库。

3. 增强图片 OCR 和有效图片筛选，只保留真正含基金信息或投放信息的图片证据。

4. 将小红书质量诊断结果进一步做成运营看板，让温姐、芬姐可以直接看到哪些博主值得跟进、哪些基金被提及、哪些内容疑似投放。

5. 持续维护项目文档、开发日志、迭代说明和入口文件，保证后续可回看、可交接、可复盘。



## 八、当前进度（重点）

基金加仓减仓看板：核心功能已上线并通过多轮校验，当前可作为正式日常观察入口。

新手导航与迭代说明：已上线，运营同事可以通过导航页了解看板入口、使用方法和历史修复内容。

小红书爬虫任务：已完成第一轮抓取流程验证和第一轮算法升级，当前已产出“质量诊断与运营分析表”。这一阶段已经能看出哪些数据可用、哪些需要补抓、哪些基金提及是确认结果、哪些是候选基金词。

小红书任务当前阶段判断：处于“算法升级与数据质量治理阶段”，尚未进入最终稳定看板阶段；下一步重点是补抓评论、优化 OCR、确认基金化名，并逐步形成运营看板。

# 小红书任务：采集口径修正与语料库接入计划（2026-04-21）

## 一、已确认的问题

当前旧采集任务中使用了 `max-notes-per-blogger=6`，这意味着每个博主最多只抓 6 条笔记。

这与业务要求“抓近两周发的帖子”不一致。

正确口径应该是：

只要帖子发布时间在近两周内，就应尽量抓取；不应被 6 条上限截断。

因此，旧长跑任务已暂停，旧结果只能作为阶段样本和算法测试数据，不能作为最终近两周完整结果。

## 二、已完成的修正

1. 暂停错误口径的旧采集进程。
2. 保留旧阶段结果，但不作为最终结果。
3. 基于 `20260420（算法迭代升级前）-标签整理_全历史池_修复清洗版_20260420.xlsx` 建立语料库。
4. 语料库已包含：
   - 基金基础标签
   - 简称/化名候选库
   - 图片 OCR 词典
   - 基金黑话方向词
   - 买卖操作/行情/广告语义词
   - 标签词频库
   - 组合识别规则

## 三、语料库作用

语料库不是最终判断结果，而是识别工具。

它用于帮助判断：

- 图片里 OCR 出来的字，是否可能是基金名称的一部分；
- 博主使用的简称、化名、行业词，是否可能指向某只基金；
- 文案或评论中是否出现买入、卖出、定投、止盈、回撤、涨跌等操作/行情信号；
- 是否出现合作、投放、链接、领取等疑似广告或转化信号。

## 四、下一步安排

### 1. 重跑采集

将采集参数调整为：

- `max-notes-per-blogger` 从 6 提高到 50 或更高；
- 继续保留 `max-note-age-days=14`；
- 使用“连续出现旧帖后停止”的逻辑控制工作量；
- 每批仍保持小批量，降低风控；
- 评论抓取继续保留，但每篇评论数先控制在合理范围。

目标：尽量抓完整每个博主近两周内的所有帖子和评论。

### 2. 接入语料库

把新语料库接入文本识别与 OCR 识别逻辑：

- 标题、正文、话题、评论：用简称/化名候选库识别基金线索；
- 图片 OCR：用 OCR 词典做纠错和模糊匹配；
- 操作/行情/广告语义词：判断内容是买卖操作、行情暗示，还是疑似广告转化。

### 3. 输出新版分析表

新版表格应增加：

- 是否命中 OCR 语料；
- 是否命中基金简称/化名；
- 是否命中买卖操作；
- 是否命中行情暗示；
- 是否命中广告信号；
- 证据片段；
- 置信度；
- 是否需要人工确认。

### 4. 人工确认闭环

把候选基金词分成三类：

- 确认基金：可进入正式基金识别库；
- 疑似简称/化名：需要业务同事确认；
- 无效词：加入排除词库。

确认后的结果会回写到语料库，下一轮识别会更准。

当前语料库已经搭好，旧采集口径的问题已经定位。

下一步核心不是继续沿用旧结果，而是用修正后的“近两周全量口径”重跑，并把语料库接入识别算法。

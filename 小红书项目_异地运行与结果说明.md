## 3. 核心运行文件

### 3.1 后端接口文件

核心后端入口：

```text
01_代码/xhs_task_dashboard/api_server.py
```

作用：

- 提供前端页面调用的本地 API。
- 负责扫码登录检测。
- 负责按博主链接抓取笔记、评论、图片等数据。
- 负责导出 Excel。
- 负责调用初步识别分析逻辑。

### 3.2 爬虫主逻辑文件

```text
01_代码/xhs_task_dashboard/scraper.py
```

作用：

- 打开小红书页面。
- 读取博主主页、笔记详情页、评论区。
- 抽取笔记文字、图片链接、评论文字、评论图片等。
- 处理登录态、页面加载、翻页、图片收集等操作。

### 3.3 前端操作页面

```text
02_前端看板/addsub/看板_小红书任务控制台.html
```

作用：

- 输入博主链接。
- 点击扫码登录/检测登录。
- 点击开始爬取。
- 查看爬取进度和分析进度。
- 导出 Excel。
- 触发识别分析。

注意：这个 HTML 页面默认调用：

```text
http://127.0.0.1:8790
```

所以必须先启动后端接口，再打开 HTML。

### 3.4 识别与增强分析文件

```text
01_代码/xhs_task_dashboard/xhs_quality_upgrade.py
01_代码/xhs_task_dashboard/ops_enrich_pipeline.py
```

作用：

- 识别基金名称、基金简称、基金标签、投资方向。
- 识别疑似广告/推广/带货信号。
- 对文本和 OCR 结果做语义增强。
- 生成更适合运营分析的表格。

### 3.5 原始素材归档文件

```text
01_代码/xhs_task_dashboard/materialize_raw_assets.py
```

作用：

- 把爬到的笔记图片、评论图片、笔记正文、评论文字等整理成可追溯的素材归档。

## 4. 在其他电脑运行的步骤

以下步骤以 Mac 为例。

### 4.1 复制项目

把桌面上的整个目录复制到新电脑：

```text
/Users/yaoruanxingchen/Desktop/xhs
```

如果新电脑用户名不同，路径可以不同，只要后续命令中的路径跟着改即可。

### 4.2 进入代码目录

假设新电脑也放在桌面：

```bash
cd ~/Desktop/xhs/快照_20260421_221835/01_代码/xhs_task_dashboard
```

如果你复制的是其他快照名，把 `快照_20260421_221835` 改成实际文件夹名。

### 4.3 安装 Python 依赖

建议使用已有 Python/Conda 环境。

```bash
python3 -m pip install -r requirements.txt
```

安装 Playwright 浏览器：

```bash
python3 -m playwright install chromium
```

如果新电脑没有 `python3`，需要先安装 Python 或 Anaconda。

### 4.4 启动后端接口

推荐使用下面这条“可迁移命令”：

```bash
cd ~/Desktop/xhs/快照_20260421_221835/01_代码/xhs_task_dashboard
export XHS_HEADLESS=false
export XHS_PROFILE_DIR="$PWD/.xhs_profile"
python3 -m uvicorn api_server:app --host 127.0.0.1 --port 8790
```

看到类似下面内容，说明启动成功：

```text
Uvicorn running on http://127.0.0.1:8790
```

注意：

- 原项目里的 `start_simple_xhs_api.sh` 写的是本机绝对路径，换电脑后可能不能直接用。
- 换电脑运行时优先使用上面的“可迁移命令”。
- 如果要使用 `start_simple_xhs_api.sh`，需要先把脚本里的路径改成新电脑路径。

### 4.5 检查后端是否正常

浏览器打开：

```text
http://127.0.0.1:8790/api/v1/health
```

如果看到 `ok:true`，说明后端正常。

也可以打开接口文档：

```text
http://127.0.0.1:8790/docs
```

### 4.6 打开前端页面

打开这个 HTML：

```text
~/Desktop/xhs/快照_20260421_221835/02_前端看板/addsub/看板_小红书任务控制台.html
```

也可以直接双击打开。

页面打开后，操作顺序是：

1. 先确认后端接口 `8790` 已启动。
2. 点击“检测登录状态”或“扫码登录”。
3. 如未登录，会弹出小红书浏览器窗口，需要扫码登录。
4. 登录完成后，回到前端页面输入博主链接。
5. 点击“开始爬取”。
6. 等待爬取进度和分析进度完成。
7. 点击导出表格。

## 9. 当前项目能产出的数据类型

当前版本主要能产出以下数据：

### 9.1 博主层数据

包括：

- 博主链接
- 博主名称
- 博主 ID
- 小红书号
- 简介
- 粉丝/关注/获赞等公开信息（视页面可见情况）
- 是否抓取成功
- 失败原因

用途：

- 判断目标博主是否可用。
- 判断博主内容风格和账号状态。

### 9.2 笔记层数据

包括：

- 笔记链接
- 笔记 ID
- 标题
- 正文
- 发布时间
- 话题标签
- 图片链接
- 视频链接（如有）
- 点赞/收藏/评论/分享等互动数据（视页面可见情况）

用途：

- 判断博主近两周内容是否涉及基金。
- 判断是否存在疑似推广、带货、软广。
- 作为后续 OCR 和语义识别的基础。

### 9.3 评论层数据

包括：

- 笔记链接
- 评论内容
- 评论作者
- 评论时间
- 是否可能是博主本人评论
- 评论图片链接（如有）

用途：

- 重点识别博主在评论区补充的基金信息。
- 识别评论区里的隐晦带货、补充链接、暗示买卖操作。

### 9.4 图片素材

包括：

- 笔记图片
- 评论图片
- 图片原始链接
- 图片本地保存路径

用途：

- 后续 OCR 识别图片文字。
- 后续识别打码、化名、基金昵称、截图式推荐信息。
- 给人工复核留证。

### 9.5 语义识别结果

包括：

- 命中的基金名称
- 命中的基金昵称/别称
- 命中的标签或投资方向
- 命中的买卖操作词
- 命中的行情暗示词
- 命中的疑似广告/推广/带货词
- 初步置信度或风险判断

用途：

- 从“爬到很多内容”转成“找到疑似基金广告线索”。
- 辅助运营判断竞品是否在小红书投放或被博主带货。

## 10. 当前项目的边界和注意事项

### 10.1 登录态不会随快照迁移

快照没有复制 `.xhs_profile` 登录缓存。

原因：

- 登录缓存包含个人登录状态，不适合打包迁移。
- 换电脑后需要重新扫码登录。

### 10.2 小红书可能触发风控

如果出现：

- 频繁要求重新登录
- 页面加载失败
- 验证码
- 抓取中断
- 空数据

通常不是单纯代码问题，也可能是平台风控、网络、账号状态、访问频率导致。

建议：

- 每次先用 1 个博主测试。
- 每个博主先抓近 20 条。
- 遇到风控后暂停一段时间。
- 不建议一次性导入大量链接猛跑。

### 10.3 图片 OCR 和语义识别仍需继续迭代

当前项目已经保存图片和评论素材，但图片中文字识别、打码内容识别、基金化名判断仍属于后续重点迭代方向。

换句话说：

- 当前已经具备“采集与留证”的基础能力。
- 当前正在推进“识别基金广告线索”的算法能力。
- 不能把初步识别结果直接当最终结论，需要人工复核。

## 11. 最推荐的异地运行验证流程

换电脑后，不要一上来跑很多博主。建议按这个顺序验证：

1. 复制 `xhs` 文件夹。
2. 安装依赖。
3. 启动后端 `8790`。
4. 打开健康检查 `http://127.0.0.1:8790/api/v1/health`。
5. 打开 HTML 前端。
6. 扫码登录。
7. 输入 1 个博主链接。
8. 抓取最新 3-5 条笔记测试。
9. 确认 Excel 能导出。
10. 确认图片、正文、评论有保存。
11. 再扩大到 20 条。
12. 再扩大到 2 个博主。
